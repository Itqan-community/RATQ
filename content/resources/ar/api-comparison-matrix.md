# مصفوفة مقارنة واجهات القرآن البرمجية

يوفر هذا المستند مقارنة جنباً إلى جنب بين أهم واجهات القرآن البرمجية وموارد البيانات المتاحة للمطورين. يغطي المصادقة، حدود الاستخدام، صيغ البيانات، وتوفر الميزات لمساعدة المطورين في اختيار المزود المناسب لمشروعهم.

للحصول على معلومات تفصيلية حول كل مزود، راجع صفحات الموارد الفردية المرتبطة في الجدول أدناه.

تركز هذه المصفوفة على الواجهات البرمجية وموارد البيانات التي توفر محتوى القرآن (نص، صوت، ترجمات، تفسير). محركات البحث فقط وأدوات التحليل (مثل alfaNous، Quran Hub API، quranic-search-v2) مغطاة في [نظرة عامة على التقنيات](../../technologies/ar/Technologies.md).

## كيفية استخدام هذه المصفوفة

- **النوع**: ما إذا كان المزود يقدم واجهة REST API مباشرة، أو تنزيلات ملفات ثابتة، أو حزمة SDK، أو مزيج منها.
- **المصادقة**: ما إذا كان التسجيل أو مفتاح API أو عدم المصادقة مطلوباً.
- **حدود الاستخدام**: سياسات حدود الاستخدام المنشورة. "غير منشور" يعني أن المزود لا يوثق حدود الاستخدام.
- **صيغة البيانات**: صيغ الاستجابة أو التنزيل المتاحة.
- **أعمدة الميزات**: ما إذا كان المزود يقدم نص القرآن، الترجمات، التلاوات الصوتية، التفسير، بيانات مستوى الكلمة، وقدرات البحث.
- **الترخيص**: شروط ترخيص استخدام الواجهة البرمجية أو البيانات. "ملكية" تعني أن المزود يتحكم في شروط الاستخدام؛ "مجاني" يعني مجاني الاستخدام بدون ترخيص رسمي مفتوح؛ التراخيص المحددة (مثل CC-BY-SA 4.0) مذكورة حيث نُشرت.

## ملخص مقارنة الميزات

### الأفضل لكل حالة استخدام

| حالة الاستخدام | المزود الموصى به | السبب |
|----------------|-----------------|-------|
| تطبيق قرآن كامل الميزات | [Quran.com API](./quran.com-api.md) أو [QUL](https://qul.tarteel.ai/) | نص + صوت + ترجمات + تفسير + بحث + بيانات مستوى الكلمة |
| الترجمات فقط | [QuranEnc](./QuranEnc.md) | 79+ لغة، مترجمون متعددون لكل لغة، صيغ تصدير متعددة |
| نص قرآني ثابت للتطبيقات غير المتصلة | [Tanzil](./tanzil.md) | نص موثق بدقة عالية بالخطين العثماني والإملائي، تنزيلات SQL/XML/TXT |
| تكامل JSON سريع | [quran-json](./quran-json.md) أو [alquran.cloud](https://alquran.cloud/) | بدون مصادقة، وصول CDN/API، صيغة JSON |
| التلاوات الصوتية | [everyayah](https://www.everyayah.com/) أو [Quran.com API](./quran.com-api.md) | everyayah لتنزيل MP3 ثابت؛ Quran.com لـ API مع طوابع زمنية |
| التفسير | [QUL](https://qul.tarteel.ai/) أو [Quran.com API](./quran.com-api.md) | QUL يحتوي على 114 تفسير؛ Quran.com يحتوي على موارد تفسير متعددة |
| بدون مصادقة | [QuranEnc](./QuranEnc.md)، [Tanzil](./tanzil.md)، [alquran.cloud](https://alquran.cloud/)، [quran-json](./quran-json.md) | جميعها مجانية بدون تسجيل |

## ملاحظات تفصيلية

### Quran.com API (مؤسسة القرآن)

- أشمل واجهة برمجية للقرآن متاحة، تشغل quran.com و QuranReflect.
- مفتاح API مجاني لكن يجب طلبه من خلال قناة Slack الخاصة بهم.
- تفريغات البيانات المصدرية غير متاحة للعموم؛ فقط المساهمون يمكنهم طلب تفريغ مصغر عبر Slack.
- الخلفية مكتوبة بـ Ruby. الكود المصدري: [quran/quran.com-api](https://github.com/quran/quran.com-api).
- البحث يستخدم بنية هجينة Elasticsearch + [Kalimat](../../technologies/ar/Kalimat.md) للبحث بالكلمات المفتاحية والدلالي.
- يوفر طوابع زمنية للصوت على مستوى الكلمة عبر Audio SDK.
- واجهات المستخدم البرمجية (الإشارات المرجعية، الملاحظات، سجل القراءة) تتطلب OAuth2.

### QuranEnc

- تديره مطبعة مجمع الملك فهد لطباعة المصحف الشريف.
- تركز الواجهة البرمجية حصرياً على الترجمات — لا تكشف عن نقاط نهاية مستقلة لنص القرآن أو الصوت. ملاحظة: يعرض موقع QuranEnc نص القرآن بجانب الترجمات في واجهة المتصفح، ومشاريع أخرى (مثل quran-json) تحصل على بيانات نصها العربي من QuranEnc، لكن نقاط نهاية الواجهة البرمجية العامة تقدم الترجمات فقط.
- الترجمات مراجعة من هيئات متخصصة وتعتبر موثوقة للغاية.
- متاح بصيغ تصدير متعددة (API, CSV, XLS, XML, PDF).
- يوفر تطبيقات جوال (Android + iOS) و PWA.

### Tanzil

- ليست واجهة برمجية مباشرة — توفر تنزيلات ملفات ثابتة فقط.
- المصدر الأكثر موثوقية لنص القرآن الموحد Unicode. العديد من المشاريع الأخرى تحصل على نسخها الصوتية وترجماتها من Tanzil.
- توفر النص بخطوط متعددة (عثماني، إملائي) مع خيارات للحركات وعلامات الوقف وأشكال التنوين.
- الترجمات متاحة بأكثر من 40 لغة بصيغ TXT, XML, و SQL dump.
- لا توفر صوتاً خاصاً بها لكن تربط بـ [everyayah](https://www.everyayah.com/) للتلاوات آية بآية.
- ليست مفتوحة المصدر.

### quran-json

- حزمة npm بواسطة [Risan Bagja Pradana](https://risanb.com) توفر نص القرآن والترجمات بصيغة JSON.
- متاحة عبر CDN (jsdelivr) للاستخدام المباشر في الويب — لا حاجة لخادم.
- تدعم ~11 لغة للترجمات. النص العربي مصدره من [QuranEnc](./QuranEnc.md)؛ النسخ الصوتي ومعظم الترجمات مصدرها من [Tanzil](./tanzil.md).
- مرخصة بموجب CC-BY-SA 4.0.

### alquran.cloud

- واجهة REST API مجانية بدون مصادقة مطلوبة لنقاط النهاية المحتوى (تم التحقق: `api.alquran.cloud/v1/ayah/1` يُعيد البيانات بدون مفتاح API). ملاحظة: [نظرة عامة على التقنيات](../../technologies/ar/Technologies.md) تذكر أنه يتطلب تسجيل — الواجهة البرمجية نفسها لا تتطلبه للقراءة.
- توفر وصولاً لنص القرآن والصوت عبر تقسيمات هيكلية متعددة (سورة، جزء، صفحة، حزب، ركوع، منزل).
- البحث متاح لكن مقتصر على النسخ النصية فقط.
- تدعم ضغط gzip و zstd للاستجابات.
- نطاقات API متعددة متاحة بما في ذلك بديل روسي.

### QUL (Tarteel)

- طوره فريق Tarteel AI. مفتوح المصدر: [GitHub](https://github.com/quran/qul).
- البديل الأغنى ميزات لـ Quran.com API: 193 ترجمة، 114 تفسير، تلاوات مع طوابع زمنية، تخطيطات مصحف، خطوط، بيانات صرفية.
- يتطلب تسجيل للحصول على مفتاح API.
- جديد نسبياً (أُطلق يونيو 2024) لكن يُصان بنشاط.

### everyayah

- يوفر ملفات صوتية ثابتة (MP3) لكل آية من القرآن.
- يوفر أيضاً صور القرآن (JPG, PNG)، نص، ملفات توقيت (XML)، وبيانات JavaScript.
- لا واجهة برمجية — الملفات تُوصل عبر روابط مباشرة.
- لا مصادقة أو حدود استخدام.
- يُستخدم من قبل Tanzil كمصدر صوتي.

### Quran-API (quranapi.pages.dev)

- واجهة برمجية مجانية بدون حدود استخدام صراحةً.
- مكتوبة بـ Python.
- توفر نص القرآن، الترجمات، وتفسير سورة كاملة أُضيف مؤخراً.
- لا مصادقة مطلوبة.

### quran-api (rzkytmgr)

- واجهة REST API مفتوحة المصدر مكتوبة بـ JavaScript.
- توفر وصولاً صوتياً كاملاً على مستوى السورة والآية.
- لا مصادقة مطلوبة.

### QuranicAudio

- مشروع مفتوح المصدر من فريق مؤسسة القرآن.
- تطبيق ويب أساسي لـ audio.quran.com يكشف أيضاً عن نقاط نهاية API.
- يركز تحديداً على التلاوات الصوتية (مستوى السورة الكاملة).
- مكتوب بـ JavaScript. المصدر: [quran/audio.quran.com](https://github.com/quran/audio.quran.com).
